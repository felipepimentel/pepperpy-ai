description: |
  A guide for enhancing task details and clarity.
  Use this prompt when:
  - Adding missing information
  - Clarifying requirements
  - Improving task descriptions
  - Adding implementation details
  - Documenting dependencies
  
  The prompt helps ensure tasks are well-documented
  and ready for execution.
---

Select a task from `/docs/status.md` to enrich:

1. Basic Information Review:
   - Task ID and title
   - Current description
   - Priority level
   - Status and timeline
   - Associated components

2. Technical Details:
   - Implementation approach
   - Required resources
   - Dependencies
   - Expected outcomes
   - Success criteria

3. Documentation Enhancement:
   - Add missing details
   - Clarify ambiguous points
   - Include examples if needed
   - Link related documents
   - Note assumptions

4. Quality Check:
   - Verify completeness
   - Check for clarity
   - Validate dependencies
   - Review acceptance criteria
   - Update timestamp

5. If Already Complete:
   - Document completeness
   - Note any observations
   - Suggest improvements
   - Update status
