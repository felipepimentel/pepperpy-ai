  A guide for identifying and resolving task blockers.
  Use this prompt when:
  - Addressing impediments
  - Resolving dependencies
  - Unblocking tasks
  - Managing bottlenecks
  - Facilitating progress
  
  The prompt helps maintain project momentum by
  systematically addressing blockers.
---

Review and resolve blockers for tasks in `/docs/status.md`:

1. Blocker Identification:
   - Review **In Progress** tasks
   - List all blockers
   - Categorize by type:
     - Technical dependencies
     - Resource constraints
     - External dependencies
     - Knowledge gaps

2. Impact Assessment:
   - Evaluate urgency
   - Identify affected tasks
   - Estimate delay impact
   - Note cascade effects

3. Resolution Planning:
   - List potential solutions
   - Evaluate alternatives
   - Identify required resources
   - Create action plan

4. Implementation:
   - Document chosen solution
   - List specific steps
   - Assign responsibilities
   - Set timeline

5. Follow-up:
   - Update task status
   - Document resolution
   - Note lessons learned
   - Monitor for recurrence
