  A systematic approach to debugging issues in the project.
  Use this prompt when:
  - Investigating reported bugs
  - Troubleshooting system errors
  - Analyzing performance issues
  - Reviewing error logs
  - Implementing fixes
  
  The prompt provides a structured methodology for identifying,
  analyzing, and resolving issues while maintaining proper documentation.
---

Debug issues in the project with the following steps:
1. Identify the source of the issue (e.g., logs, error messages).
2. Suggest potential fixes or improvements.
3. Update `.product/status.md` with the debug progress and outcomes.
