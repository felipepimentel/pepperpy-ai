  A systematic guide for reviewing and adjusting task priorities.
  Use this prompt when:
  - Reviewing project priorities
  - Adjusting task importance
  - Responding to changes
  - Balancing workload
  - Updating project focus
  
  The prompt helps maintain alignment between tasks
  and current project goals.
---

Review and adjust task priorities in `/docs/status.md`:

1. Current State Analysis:
   - List all tasks by status
   - Note current priorities
   - Check dependencies
   - Review deadlines

2. Priority Assessment:
   - Evaluate business impact
   - Consider technical dependencies
   - Check resource availability
   - Assess urgency vs importance

3. Adjustment Criteria:
   - Project goals alignment
   - Technical prerequisites
   - Resource constraints
   - Timeline requirements

4. Update Priorities:
   - Adjust priority levels
   - Document rationale
   - Update timestamps
   - Note any dependencies

5. Communication:
   - Summarize changes
   - Highlight major shifts
   - Document impact
   - Update stakeholders
